🏗️ Architecture Layers
Entry Point InitManager
앱 초기화 진입점. 모든 Manager의 초기화를 순차적으로 조율하며 Progress 추적 제공. Async/Await 기반.
▼
Manager Layer 15 Singletons
게임의 핵심 로직을 담당하는 싱글톤 매니저들. DontDestroyOnLoad로 씬 전환 시에도 유지. 각자의 도메인(UI, Sound, Pool, Save 등)을 관리.
▼
Event Bus Observer Pattern
매니저 간 느슨한 결합(Loose Coupling)을 위한 이벤트 버스. Priority 기반 실행 순서 보장. Subscribe / Unsubscribe / Publish.
▼
Data Layer JSON + ScriptableObject
JSON 파일 기반 데이터 로딩 (mapData, enemyData, playerData) + ScriptableObject로 설정 데이터 관리. Addressables 라벨로 비동기 로드.
▼
Object Pooling PoolManager + Addressables
적, 투사체, 이펙트, 맵 오브젝트의 풀링을 통한 성능 최적화. Addressable 기반 Lazy Loading + HashSet 추적. Async Prewarming 지원.
▼
UI Layer MVP Pattern
Presenter가 로직 담당, View(UI_*)가 화면 표시 담당. Popup 시스템 (GameOver, SkillChoice, MapClear). Lobby / InGame / Loading 분리.

🧩 Design Patterns
🔒 Singleton Pattern
모든 매니저가 Thread-safe Singleton 사용. DontDestroyOnLoad로 씬 전환 시 유지.
public static [Manager] Instance { get; private set; } if (Instance == null) { Instance = this; DontDestroyOnLoad(gameObject); }
📡 Observer (Event Bus)
EventType enum으로 타입 안전성 확보. Priority 기반 실행 순서. Snapshot 기반 안전한 순회.
EventBus.Subscribe(EventType.LevelUp, OnLevelUp, priority: 0); EventBus.Publish(EventType.LevelUp); EventBus.Unsubscribe(EventType.LevelUp, OnLevelUp);
🏭 Factory + Object Pool
PoolManager가 Factory 역할. Addressable 기반 Lazy Loading + HashSet 추적으로 효율적 관리.
// Addressable 기반 Lazy Loading await PoolManager.Instance.GetAsync<Enemy>(key); PoolManager.Instance.Return(enemy);
🖥️ MVP (Model-View-Presenter)
UI에 MVP 패턴 적용. Presenter가 로직 담당, View(UI_*)가 표시 담당.
// Presenter: 로직 SkillViewPresenter → 스킬 데이터 가공 // View: 표시 UI_SkillView → 화면에 렌더링
⚡ Async/Await
Unity.Awaitable 기반 비동기 초기화. CancellationToken으로 씬 전환 시 취소 지원.
async Awaitable Initialize() { await DataManager.Instance.LoadAsync(); await PoolManager.Instance.PrewarmAsync(); }
🧱 Strategy (Enemy Behavior)
IEnemyBehavior 인터페이스로 적 행동 전략 패턴 구현. Factory가 행동 모듈 조립.
interface IEnemyBehavior EnemyBehaviorFactory.Create(data) // Move: Follow, Random, Pattern // Attack: Melee, Shoot, FollowMelee
